# 工地監工 Pro — 使用說明

## 架構（手機為主、電腦為輔）

- **文字紀錄**（日期、案場、工種、廠商、狀態、備註）存在 **Supabase**（雲端資料庫）
- **照片**（原圖＋標註圖）存在你自己的 **Google Drive**
- App 部署在 **Streamlit Community Cloud**，有固定網址，手機用行動網路（4G/5G）
  隨時打開就能記錄，不需要連 Wi-Fi、不需要電腦開機
- 電腦（Mac）打開同一個網址，也能看到跟手機一模一樣的資料，適合整理、篩選、匯出報告

## 第一次設定

需要先完成 Supabase、Google Drive OAuth 授權設定，並部署到 Streamlit Community Cloud，
才能拿到手機能用的固定網址。完整步驟見同資料夾的 **`Supabase設定步驟.md`**。

## 日常使用

1. 手機瀏覽器打開你的 App 網址（建議加到主畫面，開起來跟一般 App 一樣）
2. 第一次使用先「新增案場」（例如案場名稱：桃園晴天大樓），之後每次打開只要點「進入」就會
   直接進到那個案場的工作區，不用每筆紀錄都重打一次案場名稱；要換案場用畫面右上角「切換案場」
3. 「拍照紀錄」分頁：填工種、廠商、狀態、備註 → 現場拍照或上傳 → 在照片上畫圈/線/方框標註缺失位置 → 「儲存紀錄」，照片會自動上傳到你的 Google Drive
4. 「篩選管理」分頁：側邊欄依廠商、狀態篩選，網格相冊瀏覽這個案場的所有紀錄，可直接更新狀態或刪除
5. 點「匯出報告」，會產生一份包含案場資訊、照片、備註、狀態的 PDF，自動備份一份到 Google Drive 的「報告」資料夾，同時可以直接下載到手機或電腦

## 照片存放位置

Google Drive 裡會自動建立（根資料夾「工地監工 Pro」是執行 `gdrive_oauth_setup.py` 時自動建立的）：

```
工地監工 Pro/
└── {案場名稱}/
    ├── {YYYY-MM-DD}/
    │   ├── YYYYMMDD_工種_廠商.jpg        ← 原圖
    │   └── YYYYMMDD_工種_廠商_標註.jpg   ← 標註圖
    └── 報告/
        └── 案場_缺失報告_YYYYMMDD_HHMMSS.pdf
```

同名照片不會覆蓋，自動加 `_2`、`_3` 後綴。用手機 Google Drive App 或電腦網頁版都能直接瀏覽。

## PDF 中文字型

本機執行（Mac/Windows）通常會自動抓到系統內建的中文字型。部署到 Streamlit Community
Cloud 的雲端版，需要在 repo 裡放一個 `packages.txt`（已提供），內容是 `fonts-noto-cjk`，
Streamlit Cloud 部署時會自動安裝這套中文字型，PDF 才能正常顯示中文。

## 注意事項

- `streamlit` 版本鎖在 1.39.0，因 `streamlit-drawable-canvas` 與 1.41+ 不相容，請勿自行升級。
- 刪除紀錄只會刪 Supabase 裡的文字資料，Google Drive 裡的照片仍會保留當備份。
- `secrets.toml`（裡面有資料庫金鑰、Google OAuth 授權金鑰）與 `client_secret.json` 絕對不要傳給別人、不要放上公開的地方；已加進 `.gitignore`，不會被誤傳到 GitHub。
